REST fixture does not prove stream health, and a browser paper flow does not prove private execution.

## P0 correctness gate

| Requirement | Unit/property | Integration/failure injection | Browser/E2E |
| --- | --- | --- | --- |
| delta-neutral sizing | random prices/steps/multipliers; residual delta within one allowed tolerance | venue filter mismatch, insufficient one-leg depth | paper entry blocked on mismatch; matched quantity disclosed |
| instrument identity | aliases, multiplier and same-ticker collision fixtures | registry partial failure/cache refresh | route/chart preserves venue and market type |
| quote quality | age/skew arithmetic and state machine | silent source, delayed leg, duplicate/regression/gap/checksum/resync | stale/skew warning and blocked alert/paper |
| funding schedule | variable intervals, horizon boundaries and sign | schedule refresh/failure/history fallback | exact settlements and estimate confidence shown |
| costs/capital | fee monotonicity, discrete funding, borrow and rebalance | partial account metadata and version migration | cost waterfall, capital and scenario table |
| ranking/truncation | expected-dollar/ROI ordering and stable ties | server filters before limit; total/truncated metadata | filters cannot hide omitted high-capacity routes |
| public resource limits | deterministic lease/circuit clock and stable snapshots | distinct work rejects without a queue; abort/failure/cooldown/half-open recovery | operator state remains a future dashboard task |
| alerts | rule+route crossing, hysteresis, cooldown | outbox retry, restart, duplicate delivery, notify failure | rule delivery status and retry state |
| paper ledger | deterministic event reduction | reload, migration, partial fill, exit depth and funding event | replay reproduces identical PnL |

## Release evidence rules

- A status becomes “current” only when the matching row has code, tests and documentation.
- A mocked feed proves deterministic application behavior, not exchange availability.
- A public canary proves only read-only connectivity at its capture time.
- Sandbox/testnet behavior does not prove production liquidity, matching or regional eligibility.
- “No failing tests” is not evidence for a requirement absent from the test suite.
