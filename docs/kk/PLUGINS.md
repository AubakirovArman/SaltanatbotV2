# Декларативті плагиндер

SaltanatbotV2 плагині — өңделетін индикаторлар мен стратегиялары бар жергілікті
JSON пакет. Бұл JavaScript extension немесе қашықтан жүктелетін module емес.

## Сенім шекарасы

- Ең көбі 5 МБ және 25 артефакт.
- Тек нақты рұқсат етілген fields қабылданады; белгісіз fields қабылданбайды.
- SHA-256 толық canonical manifest өзгермегенін тексереді. Қолтаңбасы бар version
  2 envelope ішінде ECDSA P-256 checksum-ға сол кілт қол қойғанын қосымша
  растайды, бірақ кілт иесінің нақты тұлғасын анықтамайды.
- `strategy_start` түбірі бар Blockly XML ғана рұқсат етіледі. `<script>` және
  arbitrary JavaScript fields қабылданбайды.
- Dependencies сол пакет ішінде болуы және self/cycle байланыс жасамауы керек.
- Қолдау көрсетілмейтін schema version және жаңа `minAppVersion` қабылданбайды.
- Import стратегияны іске қоспайды және live trading ашпайды. Әр артефакт қалыпты
  compile, validation, backtest және run-readiness кезеңдерінен өтеді.

Қолдау көрсетілетін permissions: `market.read`, `chart.overlay`, `trade.intent`
және `alert.emit`. Indicator үшін `chart.overlay`, strategy үшін `trade.intent`
қажет. Олар network, filesystem, credentials немесе тікелей exchange API қол
жетімділігін бермейді.

## Пакет жасау

Strategy Studio ішінде **Плагин құрастыру** әрекетін таңдаңыз. Пакет ID,
semantic version, лицензия және publisher деректерін толтырып, жергілікті
индикаторлар мен стратегияларды белгілеңіз. Құрастырушы барлық транзитивті
тәуелділіктерді автоматты қосады, тұрақты package-local IDs жасайды, ең аз
permissions жинағын есептейді және checksum қорғалған `.saltanat-plugin` файлын
жүктейді. Міндетті емес publisher URL тек HTTPS болуы керек. Бос таңдау, жоқ
тәуелділік және цикл қабылданбайды.

Алғашқы қолтаңбалы экспортта пайдаланушы жергілікті P-256 identity жасауды анық
растайды. Жеке `CryptoKey` экспортталмайтын түрде қайта импортталып, IndexedDB
ішінде сақталады; пакетке де, localStorage-ға да жазылмайды. Келесі пакеттерге
әдепкіде осы кілтпен қол қойылады. Үйлесімділік үшін қолтаңба checkbox-ын алып,
қолтаңбасыз version 1 envelope экспорттауға болады.

Сайт деректерін тазалау identity-ді қайтарымсыз жояды. Жеке кілтті export,
recovery және үнсіз rotation осы нұсқада әдейі берілмейді.

Автоматтандыру үшін `@saltanatbotv2/plugin-core` ішіндегі `encodePluginFile()`
қолдануға болады.

## Тексеру және импорттау

**Плагин** әрекетін таңдап, файлды ашыңыз. Техникалық тексеруден кейін міндетті
review терезесі ашылады; жергілікті кітапхана әлі өзгермейді. Пакет пен нұсқаны,
publisher, лицензияны, қолданбаның ең төменгі нұсқасын, толық checksum,
permissions және тәуелділіктері бар әр артефактті тексеріңіз. Тек **Тексерілген
плагинді импорттау** батырмасы мазмұнды қосады. Болдырмау немесе `Escape`
кітапхананы өзгертпейді. Import кезінде IDs және dependencies жаңа жергілікті
IDs-ке ауысады; plugin ID, version, publisher және checksum provenance ішінде
сақталады.

Қолтаңбалы пакет үшін review қолтаңбаның жарамдылығын, толық 64 таңбалы
fingerprint және жергілікті trust күйін көрсетеді. Жарамды, бірақ белгісіз кілтті
пайдаланушы fingerprint-ті тәуелсіз арнамен салыстырып бекітпейінше, ол сенімсіз
болып қалады. Қолтаңбасыз version 1 пакеттері қолдау табады және анық белгіленеді.
Каталог fingerprint пен import кезіндегі trust күйін сақтап, кілтке жергілікті
сенуге немесе сенімді жоюға мүмкіндік береді.

## Орнатылған пакеттер каталогы

Strategy Studio ішіндегі **Орнатылған плагиндер** батырмасы әр жергілікті
орнатуды көрсетеді: пакет ID мен нұсқасы, publisher HTTPS сілтемесі, лицензия,
қолданбаның ең төменгі нұсқасы, орнату уақыты, permissions, толық checksum,
артефактілер және өзгертілген артефактілер саны. Қайталанған импорттар бөлек
орнатулар ретінде сақталады. Ескі импорттарда жоқ metadata өрістері анық
белгіленеді.

Жою бөлек растауды талап етеді және пакеттің жергілікті артефактілерін, нұсқалар
тарихын және сақталған параметр мәндерін өшіреді. Пакет индикаторына сыртқы
кітапхана артефакті тәуелді болса, сол тәуелділік алынғанға дейін жою бұғатталады.
Өзгерістер қажет болса, алдын ала резервтік пакетті экспорттаңыз.

Кітапханадан жою іске қосылған paper/live ботты тоқтатпайды және графикке
қолданылған қабатты алып тастамайды. Бұл runtime snapshot нысандары өз
бөлімдерінде басқарылады.

## Қолтаңба шектеулері

- Self-signed кілт кілт сабақтастығын дәлелдейді, бірақ заңды атауды, domain
  иесін немесе автор беделін растамайды.
- Экспортталмайтын кілт кездейсоқ шығаруды азайтады, бірақ XSS, зиянды extension
  немесе бұзылған browser/OS шабуылынан қорғамайды.
- Trust тек осы browser profile ішінде сақталады және sync жасалмайды.
- Backup, recovery, revocation, расталған rotation және moderation registry —
  болашақ жұмыс; жаңа кілт жаңа сенімсіз fingerprint ретінде көрсетіледі.

Marketplace, URL арқылы install, executable hooks, third-party UI, auto-update,
identity authority және moderation publisher registry әзірге әдейі жоқ.
