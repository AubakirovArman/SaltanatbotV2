# PWA арқылы файл ашу

Орнатылған desktop PWA SaltanatbotV2 қолданбасын үш зерттеу форматы үшін қосымша операциялық жүйе
өңдеушісі ретінде тіркей алады:

| Кеңейтім | Мақсаты | Ең үлкен өлшемі |
| --- | --- | ---: |
| `.pine` | Жергілікті түрлендіруге арналған Pine бастапқы коды | 1 МБ |
| `.strategy` | Checksum қорғалған өңделетін артефакт | 2 МБ |
| `.saltanat-plugin` | Декларативті плагин пакеті | 5 МБ |

Бұл интеграцияны браузер мен операциялық жүйе ұсынады және бөлек рұқсат сұрауы мүмкін. Қазір ол
орнатылған Chromium-family desktop PWA үшін progressive enhancement болып табылады. Firefox, Safari,
жай браузер беті және қолдау көрсетілмейтін жүйелерде Strategy Studio ішіндегі **Pine**, **Import**
және **Plugin** қолмен таңдау жолдары толық сақталады.

## Импортқа дейінгі міндетті тексеру

Жүйе файлды ашқанда Strategy Studio файл атын, түрін және өлшемін көрсетеді. Бұл кезде қолданба
`File` объектісін алғанымен, мазмұнын әлі оқымайды. **Бас тарту** launch әрекетін толығымен жояды.

**Файлдарды жергілікті тексеру** әрекетінен кейін:

- Pine converter ішіне жүктеледі, бірақ **Түрлендіру** және **Қосу** бөлек орындалуы керек;
- `.strategy` ресурс шектерінен, schema және checksum тексеруінен өтеді, содан кейін аты, түрі,
  semantic version, schema және dependency саны көрсетіледі;
- plugin checksum, қолтаңба, signer continuity, permission, dependency және package content
  тексерулерінен өтеді.

Ашылған файл backtest, bot, paper session немесе live order іске қоспайды. Мазмұн браузерде қалады
және backend не биржаға жіберілмейді. Тек дәл кеңейтімдер қабылданады; generic `.json`, қос
кеңейтімдер, аты сәйкес емес, оқылмайтын және тым үлкен файлдар қабылданбайды. Бір launch ең көбі он
файлды қабылдайды.

## Тексеру

- `npm run pwa:check` үш нақты MIME/extension келісімін бекітіп, generic JSON және trading handler-ге
  тыйым салады.
- Unit test feature detection, өлшем шектері, spoofing және алғашқы экранда мазмұн оқылмайтынын
  тексереді.
- Production Chromium E2E Pine, strategy және plugin review/import жолдарын толық өтіп, нақты
  растауға дейін library өзгермейтінін дәлелдейді.

Жаңартудан кейін операциялық жүйе association көрсетпесе, PWA-ны қайта орнатыңыз: `file_handlers`
өзгергенде браузер тіркеуді жаңартып, рұқсатты қайта сұрауы мүмкін.
