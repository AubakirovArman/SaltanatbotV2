# Strategy Studio, Pine Script және backtest

Strategy Studio блоктардан индикатор/стратегия құруға, Pine Script-тің қолдау көрсетілетін бөлігін
импорттауға, тарихи деректермен тексеруге және параметрлерді зерттеуге мүмкіндік береді.

## Жұмыс тәртібі

1. **Стратегия** бөлімін ашып, жаңа артефакт, template, `.strategy` немесе Pine Script таңдаңыз.
2. Блоктарды біріктіріп, JSON preview мен diagnostics-ті тексеріңіз.
3. Нарық, биржа, timeframe, candle саны, капитал және шығын моделін таңдаңыз.
4. Backtest іске қосып, assumptions, data provenance, metrics, trades және equity curve қараңыз.
5. Сандық inputs болса, optimizer және walk-forward қолданыңыз.

Оң жақ панель **Build**, **Validate**, **Preview**, **Backtest**, **Optimize**, **Run** және
**Learn** кезеңдеріне бөлінген. Validation diagnostic нақты block-ты таңдайды, ал Learn бөлімі
оның description, inputs, output, example және pitfalls мәліметін көрсетеді. Үш қадамды wizard
EMA-cross, RSI немесе breakout логикасын кәдімгі өңделетін Blockly XML ретінде жасайды.

Сандық parameter default/min/max/step және optimization eligibility сақтайды. Пайдаланушының
Blockly function-дары numeric arguments алмастыру арқылы compile-time subgraph ретінде ашылады;
recursion және шектен тыс терең шақыру қабылданбайды.

## Version және portable файл

Әр мағыналы save semantic version, logic hash, immutable алдыңғы revision, parameter schema және
indicator dependencies сақтайды. Version панелі diff көрсетеді және бұрынғы нұсқаға жаңа current
revision жасау арқылы rollback орындайды. Missing және cyclic dependency іске қосуға дейін көрінеді.

Schema-v2 `.strategy` файлында schema/artifact version, SHA-256 checksum, IR hash, parameters,
dependencies және provenance бар. Import checksum-ды алдымен тексереді; legacy v1 explicit migration
арқылы өтеді. URL share ыңғайлы, бірақ signed немесе trusted package емес.

## Pine импорты

Диалог мәтінді немесе 25-ке дейін `.pine` файлын қабылдайды. `indicator()` индикаторға,
`strategy()` стратегияға айналады. Pine v4–v6 танылады, алайда нақты үйлесімділік нұсқа нөмірімен
емес, іске асырылған конструкциялармен анықталады.

- exact конструкциялар блоктар мен IR-ге айналады;
- approximations ескерту береді;
- қауіпсіз candle execution мүмкін емес конструкциялар себеппен қабылданбайды;
- техникалық identifiers пен Pine коды аударылмайды.

Импорттан кейін Studio бастапқы Pine кодын, өңделетін blocks және compiled
preview-ді қатар көрсетеді. Pine source импорт дәлелі ретінде өзгермейді: blocks
өңдеу оны қайта жазбайды. Diagnostic code батырмасы тиісті source range-ті
таңдап, оған focus береді.

Ескерту TradingView-пен эквиваленттілікті растамайды. Бірдей candles қолданып сигналдарды салыстырыңыз.

## Нәтижені түсіндіру

Backtest execution model, комиссия, slippage, funding, warm-up және position sizing-ке тәуелді.
Report деректердің шығу тегін және versioned traces көрсетеді. Mixed, synthetic, fallback немесе
unknown деректер performance claim үшін жарамсыз. Monte Carlo — болжам емес, іске асқан trades
ретін өзгерту арқылы robustness зерттеуі.

### Портфель бэктесті

**Портфель режимін** қосып, екіден алтыға дейін әртүрлі нарықты және бір мезгілдегі
позициялар, жалпы экспозиция, бір позиция экспозициясы мен ең аз ішінара бөлу
лимиттерін таңдаңыз. Studio әр нарық үшін мәміле кандидаттарын жасап, оларды уақыт
бойынша бір ортақ капитал арқылы өткізеді. Есеп портфель equity curve, drawdown,
exposure, funding, қабылданбаған кірулер, нарық үлесі және synchronized return
correlation matrix көрсетеді.

Тәуекел зертханасы ортақ equity returns негізінде тарихи 95%/99% VaR және
expected shortfall, ең нашар кезең, Ulcer индексі, ең ұзақ recovery және бөлінген
номинал concentration есептейді. Детерминирленген moving-block bootstrap қысқа
volatility кластерлерін сақтау үшін үздіксіз return блоктарынан 1000 траектория
құрады. Есеп P5/P50/P95 пайда мен drawdown, шығын, капиталдың жартысын жоғалту
және ruin ықтималдығын көрсетеді. Бұл болжам немесе кепілденген шығын шегі емес.

Stress matrix қабылданған мәмілелерді төрт сценарийде қайта бағалайды: кіру мен
шығудың әрқайсысына қосымша 5 bps, әр шығуға 25 bps нашарлау, байқалған funding
екі еселенуі және combined shock. Қосымша шығындар ортақ equity curve-ке шығу
уақытында жазылып, stressed net profit пен max drawdown көрсетіледі. Break-even
buffer оң базалық пайданы жоятын әр орындауға қосымша bps мөлшерін береді. Бұл
order-book capacity, жаңа сигналдар немесе басқа болашақ market path моделі емес.

Орындау сапасы панелі қабылданған портфель мәмілелерінің транзакциялық шығындарын
детерминирленген түрде бөледі. Комиссия мәміленің gross және net PnL мәндерінен
салыстырылады, funding execution trace ішінен алынады, ал бапталған қолайсыз
slippage симулятор оны қолданатын fill жолдарында ғана кері есептеледі. Есеп
шығындарға дейінгі gross PnL, толық bps, нәтижеге әсер және нарық пен шығу себебі
бойынша кестелерді көрсетеді. Бұл модельдік TCA; order book impact, latency немесе
venue quality telemetry өлшемі емес.

V1 шекарасы: кандидаттар алдымен жеке market backtest арқылы есептеледі, кейін
ортақ портфель олардың көлемін өзгертеді немесе кіруді қабылдамайды. Сондықтан
strategy equity оқитын сигнал әзірге бүкіл портфель капиталын емес, сол нарықтың
жергілікті капиталын көреді. Бұл assumption есеп пен JSON export ішінде сақталады.

Әр нәтиже immutable symbol/timeframe/exchange, market/price type, data range,
execution settings, fill assumptions және missing/partially-loaded history
мәліметтерін сақтайды. Export батырмасы versioned `.saltanat-report.json`
жасайды. Settings, range, quality немесе provenance әртүрлі runs салыстырылмайды.

Bar replay previous/next батырмалары және range control арқылы deterministic
қадам жасайды. Әр frame strategy/broker events, equity, expression explanations
және variable changes көрсетеді.

Walk-forward екі режим ұсынады: rolling тәуелсіз windows қолданады, anchored
бірінші candle-дан training range-ті кеңейтеді және disjoint OOS folds тексереді.
Parameter stability view winning values диапазонын көрсетіп, unstable параметрді
бөлек белгілейді.

## Генератор кандидаттарын серверде бағалау (R9.1, қабылданды)

R9.1 — **қабылданды және өндіріске шықты**; жазылған
[R9.1 acceptance evidence](../evidence/R9_1_SERVER_EVALUATION.md) құжатын қараңыз.

Артефакт кітапханасындағы стратегия генераторы құрылымдық кандидаттар жасайды,
бірақ өзі свеча жүктемейді және backtest қоспайды. **«Серверлік бағалау (көп
нарық)»** бөлімі таңдалған кандидатты серверге зерттеу тапсырмасы ретінде
жібереді: бір таймфреймді ортақ қолданатын бірден алтыға дейін нарық, тарих
тереңдігі (әдепкі 3000 бар), train үлесі (әдепкі 0.7) және эмбарго (әдепкі
8 бар) таңдалады. Сервер тек нақты жабылған биржа барларын қолданады —
синтетикалық деректерге тыйым салынған; тарих жеткіліксіз болса, тапсырма
нақты себеппен қатеге аяқталады.

Деректер dataset-v1 fingerprint арқылы бекітіледі; train және out-of-sample
терезелері эмбарго аралығымен бөлінеді, сондықтан болашаққа қарау мүмкін емес.
Әр нарық үшін train және OOS backtest, содан кейін OOS терезелерінде ортақ
капиталмен портфель прогоны орындалады. Дайын нәтижелер таза ранкерге беріледі:
рейтинг бөлімі айыппұлдар бөлінісін, dataset fingerprint пен қозғалтқыш
нұсқасын көрсетеді. Бір иеленушіге ең көбі бес белсенді серверлік тапсырма
квотасы қолданылады; тапсырма күйлері (кезекте, орындалуда, қате, тоқтатылды)
айқын көрсетіледі.

Бұл тек зерттеу: жоғары рейтинг ештеңе іске қоспайды және жарияламайды.
Импортталған кандидат бұрынғыдай кәдімгі backtest, walk-forward және paper
тексеруінен өтеді. Бағалау әлемі — қазіргі публикалық каталог қана, сондықтан
нәтижелерде survivorship bias бар.

## Серверлік GA эволюциясы және промоушен (R9.2, қабылданды)

R9.2 — **қабылданды және өндіріске шықты** (аддитивті PostgreSQL 16→17
ga_evolution_lineage миграциясы); жазылған
[R9.2 acceptance evidence](../evidence/R9_2_GA_EVOLUTION.md) құжатын қараңыз.
Бұл бөлімдегінің бәрі — тек зерттеу; версияланған стратегия галереясы — келесі
бөлімде сипатталған жеке қабылданған R9.3 инкременті.

Генератор панелінің **«Серверлік эволюция (GA)»** бөлімі серверлік эволюциялық
прогонды іске қосады: ортақ таймфреймі бар бірден төртке дейін нарық, тарих
тереңдігі, эмбаргосы бар train/OOS бөлінісі, seed, 8–64 популяция және 1–16
ұрпақ. Сервер нақты жабылған барларды бір рет жүктеп, dataset-v1 fingerprint
арқылы бекітеді және ұрпақтарды seed бойынша детерминирленген кездейсоқ сандар
генераторымен өсіреді. Кандидаттар fingerprint бойынша дедупликацияланады және
ешқашан қайта есептелмейді; әрбір жаңа кандидат әр нарық үшін train/OOS
backtest пен OOS терезелерінде ортақ капиталды портфель прогонынан өтіп,
мақсаттар векторын (әдепкі: таза пайда — максимум, drawdown — минимум, Sharpe —
максимум, құрылымдық күрделілік — минимум), шежіресін (ата-аналар мен мутация
журналы) және әр мақсат бойынша train-мен-OOS алшақтығы, шығынды OOS үлесі,
нарықтар арасындағы шашырау және айқын overfit/unstable белгілері бар OOS
есебін алады. Парето рангтары кумулятивті қайта есептеледі; ранг 0 — прогон
кестесіндегі фронтир.

Әр ұрпақ чекпоинтпен (популяция геномдары мен генератор күйі) бірге атомарлы
сақталады: «Тоқтату» прогонды сәтсіздік емес, checkpointed күйінде аяқтайды, ал
«Жалғастыру» оны дәл жалғастырады — бірдей seed және датасет үзіліске
қарамастан бірдей кандидаттарды, мақсаттарды және шежірені береді; бұл R9.2
релизінің критерийі. Жалғастыру кезінде деректер қайта жүктеліп, fingerprint
салыстырылады; сәйкессіздік болса, прогон ga_dataset_drift қатесімен айқын
аяқталады — детерминизм ешқашан үнсіз бұзылмайды. Бір иеленушіде бір мезгілде
ең көбі бір белсенді GA прогоны болады.

«Промоушен» фронтир кандидатын тек **өз** стратегия кітапханаңызға кәдімгі
портативті артефакт ағыны арқылы көшіреді; шығу тегі жазылады: прогон,
fingerprint, seed, датасет fingerprint, қозғалтқыш/генератор нұсқалары, шежіре
және OOS есебі. OOS есебі жоқ немесе overfit белгісі бар кандидатқа сервер бас
тартады, ал интерфейс әрекетті сол себеппен өшіреді. Промоушен жасалған
артефакт — кәдімгі өңделетін стратегия: ол ештеңе іске қоспайды; backtest,
walk-forward және paper тексеруі бұрынғыша жалғыз жол.

## Версияланған стратегия галереясы (R9.3, қабылданды)

R9.3 — **қабылданды және өндіріске шықты** (аддитивті PostgreSQL 17→18
versioned_strategy_gallery миграциясы) және R9-ды аяқтайды; жазылған
[R9.3 acceptance evidence](../evidence/R9_3_STRATEGY_GALLERY.md) құжатын
қараңыз. Галерея стратегияны сол self-hosted инстанцияның басқа аккаунттарына
жариялауға мүмкіндік береді және үш кепілдік ұстанады: жарияланған артефакт
бекітілген нұсқалар бойынша қайта жаңғыртылады, иеленуші деректерін ашпайды
және импорттан кейін білдірмей өзгере алмайды.

**Жариялау.** Артефакт кітапханасының **«Галерея»** бөлімі кітапхана артефактын
(IR және өзі мәлімдеген нарықтар/метрикалар) немесе **промоушен жасалған**
GA-кандидатты (серверлік train/OOS дәлелдері, датасет fingerprint, seed және
қозғалтқыш/генератор нұсқалары) жариялайды. Жіберер алдында диалог
**санитизация алдын ала қарауын** — хешталып сақталатын дәл сол канондық
JSON-ды — көрсетеді және айқын келісім белгісін талап етеді; мазмұн өзгерсе,
келісім кері қайтарылады. Серверлік санитайзер тек ақ тізімдегі өрістерді
көшіреді және сериялданған бандлде иеленуші, прогон немесе тапсырма
идентификаторы кездессе, жариялаудан бас тартады: жалдаушы идентификаторлары,
workspace сілтемелері мен шежіре деректері сақталған артефактқа ешқашан
кірмейді. Кітапхана метрикалары **self-reported** (расталмаған мәлімдеме)
белгісімен, GA метрикалары серверлік out-of-sample дәлел ретінде сақталады;
екеуі де міндетті шектеулер ескертпесін алады.

**Өзгермейтіндік және хеш.** Жариялау — өзгермейтін `(id, нұсқа)` жолы; сол id
бойынша қайта жариялау алдыңғыларын өзгертпей келесі нұсқаны жасайды. Мазмұн
дерекқор триггерімен қатырылған — тек көріну мен кері қайтару күйі өзгере
алады — және қасында бандлдің канондық JSON бойынша SHA-256 хеші сақталады.
Дәл сол канондық алгоритм браузерде де жұмыс істейді, сондықтан импорт кезінде
хеш **екі жақта да** тексеріледі: сервер сәйкес келмейтін хешпен бандл
бермейді, ал клиент хешті өзі қайта есептеп, сәйкессіздікті айқын қатемен
қабылдамайды.

**Рейтинг.** Лента карточкасы рейтингті **тек бөліністерімен көрсетеді — жалаң
балл да, жалаң табыс та ешқашан емес**: пайда әдейі компонент емес. 0–100
композиті OOS тұрақтылығын (0.35; overfit белгісінде нөл, unstable кезінде
жарты, OOS бөлімі жоқ self-reported метрикада нөл), drawdown-ды (0.25),
қайта жаңғыртылуды (0.2 — датасет fingerprint, seed, қозғалтқыш пен генератор
нұсқалары), құрылымдық күрделілікті (0.1) және дәлел жаңалығын (0.1, 365 күнде
азаяды) өлшейді. Карточка сондай-ақ in-sample мен out-of-sample метрикаларын,
айқын overfit/unstable белгілері бар OOS алшақтығын, шектеулерді және айқын
«кері қайтарылған» күйін көрсетеді.

**Импорт.** Импорт тексеру диалогын (`.strategy` файлын тексеру үлгісімен)
ашады: расталған хеш, шығу тегі, self-reported ескертуі және шектеулер
ескертпесі. Растау кітапханада «галерея» шығу тегі бар **жеке тәуелсіз көшірме**
жасайды — жергілікті кітапхана серверлік жолға ешқашан байланбайды — және оны
«қайта валидация қажет» деп белгілейді: дәл осы көшірмеде жергілікті тексеру
мен backtest аяқталғанша paper іске қосу құлыптаулы қалады. Жариялау мен импорт
ешқашан робот қоспайды. Өз жариялауын кері қайтару айқын себепті талап етеді,
барлық нұсқаларды кері қайтарылған деп белгілейді (импорт айқын қатемен
қабылданбайды) және мазмұнды ешқашан қайта жазбайды; иеленуші өзінің кері
қайтарылған тарихын оқи алады. Көрінуді (private/unlisted/public) кез келген
уақытта өзгертуге болады, ал бүкіл галерея, публикалық лентаны қоса, pre-HTTPS
орналастыруда сессиямен қорғалған күйде қалады.

Live-ке дейін бірнеше нарық/кезеңді, out-of-sample нәтижені және paper журналын тексеріңіз.
