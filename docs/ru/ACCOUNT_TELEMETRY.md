# Телеметрия экономики аккаунта

Статус: защищённый read-only P0 API для Binance и Bybit реализован. Это источник данных для
исследования и оператора, а не разрешение на торговлю и не заявление о готовности к mainnet.

## Доступ

`GET /api/trade/account-telemetry` находится внутри существующей торговой сессии и требует роль
`admin`. Ключи берутся только из зашифрованных настроек сервера. Передать ключ через запрос нельзя;
в ответе нет ключей, подписей и секретов. Все запросы к биржам — только GET, поэтому включать live
trading для чтения не требуется.

Параметры жёстко ограничены: `venues` — Binance/Bybit (до двух), `symbols` — 1–2 символа,
`assets` — 1–4 актива, `stableAssets` — 1–3 актива кроме USDT. По умолчанию используются
`BTCUSDT,ETHUSDT`, `BTC,USDT,USDC` и `USDC`.

## Что возвращается

- Binance: подписанные компоненты spot-комиссии, текущая ставка и tier USDⓈ-M, статус BNB fee burn,
  доступный cross-margin заём и следующая почасовая ставка, а также статусы/комиссии сетей ввода и
  вывода.
- Bybit: подписанные ставки Spot/Linear, `availableToBorrow` и почасовая ставка UTA, статусы и
  комиссии chain для ввода/вывода.
- Stablecoin FX: bid/ask Binance и Bybit. У Binance REST `bookTicker` нет времени биржи, поэтому его
  receive-time запись показывается как provenance, но сама по себе не разрешает economic readiness.
  У Bybit используется время из envelope.

Каждое доказательство содержит `source`, `version`, `asOf`, `validUntil`, `timestampQuality` и
`fresh`; срок действия — 30 секунд. Старый или некорректный ответ не подменяется последним удачным.

## Интерфейс

Администратор может открыть **Торговля → Настройки → Экономика текущего аккаунта** и вручную
обновить тот же ограниченный snapshot. Панель EN/RU/KK повторно проверяет контракт в браузере,
отклоняет небезопасные readiness-флаги, пересечённый FX-стакан, неизвестную версию и слишком большие
массивы. Комиссии, займ, сети и FX показаны семантическими таблицами. Snapshot не сохраняется в
browser storage, кнопок выставления ордера нет, для не-admin сессии панель не отображается.

## Почему `executable` остаётся false

- Будущий asset комиссии зависит от фактического исполнения. BNB показывается только как условный
  discount asset, а окончательный asset подтверждает authenticated fill.
- API бирж сообщает текущую доступность и ставку займа, но не доказывает, что займ нельзя отозвать.
  Поэтому `recallStatus=unknown`, а стратегии с требованием non-recallable borrow блокируются.
- Сеть считается пригодной только при свежих данных, включённых deposit+withdraw и известной
  комиссии. Дополнительно необходимо сопоставить одну и ту же сеть на обеих биржах.
- Этот API не заменяет sequence-verified стакан, резерв капитала, подтверждение прихода перевода и
  учёт фактических fills.

I/O ограничен: одна приватная загрузка на биржу, не более трёх внутренних запросов одновременно,
таймаут 5 секунд на fetch и 20 секунд на весь refresh, ограниченный размер ответа. После трёх полных
сбоев circuit открывается на 30 секунд. Частичный результат имеет статус `partial` и список `issues`.
