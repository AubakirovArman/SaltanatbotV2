# Межбиржевой арбитражный скринер

Раздел **Скринер** без API-ключей проверяет общие USDT-пары Binance и Bybit. Для каждой пары
рассчитываются два направления:

1. купить spot Binance по ask и открыть short бессрочного linear-фьючерса Bybit по bid;
2. купить spot Bybit по ask и открыть short бессрочного USD-M-фьючерса Binance по bid.

Скринер показывает грязный спред, издержки по отдельным профилям комиссий, чистый спред, доступный объём
в лучших ценах и funding. Объём ограничен меньшей из двух сторон.
Разница больше 20% по модулю отбрасывается: это чаще коллизия одинаковых тикеров разных активов,
реденоминация или устаревший рынок, а не исполнимая возможность.

```text
грязный спред (bp) = (bid фьючерса - ask спота) / ask спота × 10 000
чистый спред (bp)  = грязный спред - оценка общих издержек
доступный объём    = min(ask спота × размер ask, bid фьючерса × размер bid)
```

В браузере отдельно задаются taker-комиссии спота и фьючерсов Binance/Bybit и запас на
проскальзывание туда-обратно. Вход и выход учитываются автоматически. Funding
показывается отдельно: заранее неизвестны срок удержания и будущие ставки.

API опрашиваются параллельно, результаты кэшируются на сервере на две секунды и передаются через
read-only WebSocket `/arbitrage-stream`. При разрыве соединение восстанавливается с ограниченной
задержкой, а REST используется как резерв. Временный полный
отказ может показать последний успешный снимок не старше 30 секунд с явной пометкой. В скрытой
вкладке браузер приостанавливает поток.

## Глубина, алерты и paper-позиции

`GET /api/arbitrage/depth` по явному запросу получает до 100 уровней каждого стакана, проходит по
ask спота и bid фьючерса для выбранной суммы и показывает VWAP, худшую цену, количество уровней,
проскальзывание и полноту двух ног. Если видимой ликвидности не хватает хотя бы в одном стакане,
paper-вход блокируется.

Алерт срабатывает только при пересечении настроенного чистого порога. Уведомление рабочего стола
требует разрешения браузера. Telegram отправляется только при авторизованной paper-trade сессии и
включённом канале уведомлений. Алерты не отправляют ордера.

Paper-позиции хранятся локально в браузере. Вход использует VWAP из двух стаканов, текущий PnL
закрывает спот по bid и short фьючерса по ask и вычитает оценку полного круга комиссий. Это не
биржевой счёт.

Скринер работает только для исследования и не отправляет ордера. Котировки двух бирж не атомарны.
Комиссии, реальная глубина, funding, доступность займа, переводы, задержка и риск ликвидации могут
полностью убрать найденную разницу. Положительный результат не является гарантией прибыли.
