# Трассировка событий стратегии

SaltanatbotV2 создаёт версионированную детерминированную трассировку для каждой вычисленной свечи. Каноническая схема и нормализатор находятся в `packages/strategy-core/trace.ts`.

## Совместимость версии 1

Каждый `StrategyBarTrace` содержит:

- `v: 1`;
- индекс свечи `barIndex` с нуля;
- время исходной свечи `barTime`;
- упорядоченный список семантических intent-событий.

V1 включает entry, exit, stop, target, trail, size, alert, marker и превышение бюджета выполнения. Нечисловые значения нормализуются в `null`, поэтому трассировку безопасно сериализовать в JSON.

Фикстура `frontend/tests/strategyEventTrace.golden.json` сохраняет эту семантическую V1-проекцию для проверки совместимости.

## Версия 2

V2 сохраняет смысл и порядок V1-событий и добавляет:

- объяснения условий, значений и границ циклов со стабильным statement path;
- тип выражения, итоговый результат и количество вычислений без повторного запуска выражения;
- `trueCount` для многократно проверяемых boolean-выражений;
- отсортированные по имени изменения переменных before/after на каждой свече;
- явные флаги усечения и ограничения по 256 объяснений/изменений переменных на свечу.

Preview, backtest и backend evaluator для paper/live получают полную V2-трассировку из общего evaluator. Cross-runtime тест сравнивает объяснения и изменения переменных для каждой свечи и одновременно защищает V1 golden-проекцию.

## Правила версионирования

- Нельзя менять смысл или порядок существующей версии.
- Ломающие изменения требуют новой версии и описания миграции.
- Нельзя добавлять wall-clock timestamps, случайные ID, локализованный текст или секреты биржи.
- Разрешены время/индекс свечи, нормализованные числа и стабильные семантические идентификаторы.

## Оставшаяся работа

Strategy V2 доказывает совпадение intents, объяснений и изменений переменных между runtime-слоями. Историческое выполнение дополнительно создаёт `BacktestExecutionTrace` v1 из `packages/backtest-core/executionTrace.ts`. Его события включают запланированные/отброшенные fills, отклонённые входы, переходы позиции/капитала, funding, коды предупреждений и итоговый provenance. Preview не имитирует отсутствующие биржевые fills, поэтому layer-specific различия остаются явными.
