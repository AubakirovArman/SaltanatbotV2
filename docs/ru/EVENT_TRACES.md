# Трассировка событий стратегии

SaltanatbotV2 создаёт версионированную детерминированную трассировку для каждой вычисленной свечи. Каноническая схема и нормализатор находятся в `packages/strategy-core/trace.ts`.

## Версия 1

Каждый `StrategyBarTrace` содержит:

- `v: 1`;
- индекс свечи `barIndex` с нуля;
- время исходной свечи `barTime`;
- упорядоченный список семантических intent-событий.

V1 включает entry, exit, stop, target, trail, size, alert, marker и превышение бюджета выполнения. Нечисловые значения нормализуются в `null`, поэтому трассировку безопасно сериализовать в JSON.

Preview, исторический backtest и backend evaluator, используемый paper/live-ботами, получают события из одного `traceBarIntents()`. Фикстура `frontend/tests/strategyEventTrace.golden.json` обязана совпадать во всех путях для каждой свечи.

## Правила версионирования

- Нельзя менять смысл или порядок существующей версии.
- Ломающие изменения требуют новой версии и описания миграции.
- Нельзя добавлять wall-clock timestamps, случайные ID, локализованный текст или секреты биржи.
- Разрешены время/индекс свечи, нормализованные числа и стабильные семантические идентификаторы.

## Оставшаяся работа

V1 доказывает совпадение intents между runtime-слоями. Следующая версия должна явно добавить объяснения выражений, изменения переменных, решения о fill, переходы position/equity, provenance данных и предупреждения. Layer-specific события должны быть объявлены явно: preview не должен имитировать отсутствующие биржевые fills.
