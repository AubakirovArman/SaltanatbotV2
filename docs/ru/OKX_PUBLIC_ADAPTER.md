# Публичный REST-адаптер OKX

Статус: read-only backend-компонент проверен 14 июля 2026 года, доступен через
`/api/market-data/okx/*` и участвует в нормализованном `/api/instruments`. Он не подключён к live
арбитражному сканеру, chart UI или приватной торговле.

Адаптер без API-ключей поддерживает публичные данные OKX API v5:

- метаданные `SPOT`, бессрочных `SWAP` и срочных `FUTURES`;
- список тикеров и top-of-book одного инструмента;
- полный REST-снимок стакана глубиной от 1 до 400 уровней;
- текущую оценку, расписание и историю funding для `SWAP`.

Нативный `instId` сохраняется без удаления дефисов. Для спота количество измеряется в базовом
активе. Для деривативов шаг лота и размеры стакана измеряются в контрактах; дополнительно
сохраняются `ctVal`, `ctMult`, `ctValCcy`, тип linear/inverse, валюта расчёта и срок экспирации.
Публичные метаданные не содержат универсальный минимальный notional для каждого продукта, поэтому
значение `minimumNotional: 0` означает «неизвестно», а не отсутствие ограничения.

Интервал funding вычисляется по `fundingTime` и `nextFundingTime`, без предположения о фиксированных
восьми часах. Ошибка текущего funding блокирует результат. Если недоступна только история, текущая
оценка возвращается вместе с явной ошибкой источника.

Каждый запрос ограничен timeout и размером ответа, поддерживает отмену и проверяет HTTP-статус,
OKX `code`, JSON и смысловые инварианты. Пересечённый top-of-book, несортированный/пересечённый
стакан, некорректные числа, timestamp или sequence приводят к безопасному отказу. Registry хранит
последний успешный срез отдельно для OKX SPOT/SWAP/FUTURES, поэтому частичный сбой одного источника
не удаляет ранее проверенные инструменты.

Пока намеренно не реализованы WebSocket и восстановление стакана, подключение к сканеру/UI,
margin/options/spreads, чтение аккаунта, займы, переводы и исполнение ордеров. Capability manifest
явно отмечает приватное исполнение как `false`.

Полное техническое описание: [OKX public adapter](../OKX_PUBLIC_ADAPTER.md). Официальный источник:
[OKX API v5](https://www.okx.com/docs-v5/en/).
