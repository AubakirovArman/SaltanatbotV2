# Декларативные плагины

Плагин SaltanatbotV2 — локальный JSON-пакет с редактируемыми индикаторами и
стратегиями. Это не JavaScript-расширение и не удалённо загружаемый модуль.

## Граница доверия

- Максимум 5 МБ и 25 артефактов.
- Допускаются только явно описанные поля; неизвестные поля отклоняются.
- SHA-256 защищает весь канонический manifest от изменения, но не подтверждает
  личность или надёжность издателя.
- Разрешён только Blockly XML с корнем `strategy_start`. `<script>` и поля с
  произвольным JavaScript отклоняются.
- Зависимости должны находиться внутри пакета и не образовывать self/cycle links.
- Новые неподдерживаемые версии схемы и `minAppVersion` отклоняются.
- Импорт не запускает стратегию и не открывает live trading. Артефакт проходит
  обычные compile, validation, backtest и run-readiness этапы.

Поддерживаемые permissions: `market.read`, `chart.overlay`, `trade.intent` и
`alert.emit`. Индикатору требуется `chart.overlay`, стратегии — `trade.intent`.
Они не дают доступ к сети, файлам, ключам или прямому API биржи.

## Создание пакета

В Strategy Studio выберите **Собрать плагин**, заполните ID, семантическую
версию, лицензию и данные издателя, затем отметьте нужные локальные индикаторы и
стратегии. Мастер автоматически добавляет все транзитивные зависимости,
назначает детерминированные ID внутри пакета, вычисляет минимальные permissions и
скачивает `.saltanat-plugin` с checksum. Необязательная ссылка издателя должна
использовать HTTPS. Пустой набор, отсутствующая зависимость или цикл отклоняются.

Для автоматизации можно использовать `encodePluginFile()` из
`@saltanatbotv2/plugin-core`.

## Проверка и импорт

Выберите **Плагин** и файл. После технической проверки открывается обязательный
экран review; локальная библиотека ещё не изменяется. Проверьте пакет и версию,
издателя, лицензию, минимальную версию приложения, полный checksum, permissions и
каждый артефакт с зависимостями. Только кнопка **Импортировать проверенный
плагин** добавляет содержимое. Отмена или `Escape` оставляют библиотеку без
изменений. При импорте ID и зависимости становятся новыми локальными ID, а plugin
ID, версия, publisher и checksum сохраняются в provenance.

Пока намеренно отсутствуют marketplace, установка по URL, исполняемые hooks,
сторонний UI, auto-update и подписи издателей. Для них сначала нужны отдельные
правила permissions, модерации, signing и supply-chain security.
