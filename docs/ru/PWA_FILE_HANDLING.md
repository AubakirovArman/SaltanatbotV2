# Открытие файлов через PWA

Установленное desktop PWA может зарегистрировать SaltanatbotV2 как дополнительный системный
обработчик трёх исследовательских форматов:

| Расширение | Назначение | Максимальный размер |
| --- | --- | ---: |
| `.pine` | Исходник Pine для локального преобразования | 1 МБ |
| `.strategy` | Редактируемый артефакт с контрольной суммой | 2 МБ |
| `.saltanat-plugin` | Декларативный пакет плагина | 5 МБ |

Интеграцию предоставляет браузер и операционная система, которые могут запросить отдельное
разрешение. Сейчас это progressive enhancement для установленных desktop PWA семейства Chromium.
В Firefox, Safari, обычной вкладке и неподдерживаемой системе остаются полноценные ручные кнопки
**Pine**, **Импорт** и **Плагин** в Strategy Studio.

## Обязательная проверка перед импортом

После системного открытия Strategy Studio показывает имя, тип и размер файла. На этом экране
приложение уже получило объект `File`, но ещё не прочитало содержимое. **Отмена** полностью
отбрасывает запуск.

После **Проверить файлы локально**:

- Pine загружается в конвертер, но отдельно требуются **Преобразовать** и **Добавить**;
- `.strategy` ограничивается по ресурсам и проверяется по schema/checksum, затем отдельный диалог
  показывает имя, тип, версию, schema и число зависимостей;
- plugin проходит существующую проверку checksum, подписи, continuity подписанта, permissions,
  зависимостей и состава пакета.

Файл не запускает бэктест, бота, paper-сессию или live-ордер. Содержимое остаётся в браузере и не
отправляется backend или бирже. Принимаются только точные расширения; generic `.json`, двойные
расширения, подмена имени, нечитаемые и слишком большие файлы отклоняются. Один запуск ограничен
десятью файлами.

## Проверка реализации

- `npm run pwa:check` фиксирует три точных MIME/extension-контракта и запрещает generic JSON/trading
  handlers.
- Unit-тест проверяет feature detection, лимиты, spoofing и отсутствие чтения содержимого на первом
  экране.
- Production Chromium E2E проходит полный review/import для Pine, strategy и plugin и доказывает,
  что библиотека не меняется до явного подтверждения.

Если после обновления системная ассоциация не появилась, переустановите PWA: браузер обновляет
регистрацию и может повторно запросить разрешение при изменении `file_handlers`.
