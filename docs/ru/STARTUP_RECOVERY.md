# Восстановление запуска приложения

SaltanatbotV2 не оставляет пользователя перед необъяснимым пустым экраном. До запуска React
`index.html` показывает минимальный локализованный экран состояния. Если React render или lazy
workspace завершается ошибкой, глобальный boundary заменяет интерфейс экраном восстановления.

Доступны три действия:

- **Повторить запуск интерфейса** — повторно монтирует React без изменения хранилища.
- **Перезагрузить страницу** — выполняет обычный reload.
- **Обновить файлы приложения** — удаляет только регистрацию собственного
  `/service-worker.js` и Cache Storage с префиксом `saltanat-shell-`, затем перезагружает страницу.

Графики, workspace, стратегии, IndexedDB signing identity, настройки бирж, журнал ботов,
localStorage и торговая база данных не очищаются. Ошибка загрузки dynamic import/chunk получает
только одну автоматическую попытку обновления shell на вкладку, поэтому бесконечный reload-loop
невозможен. Обычные ошибки приложения требуют явного действия пользователя.

Если экран остаётся после обновления файлов, проверьте ответы `200` для HTML, startup assets и
content-hashed `/assets/*.js`, корректный JavaScript MIME type, CSP и cache headers reverse proxy.
Production E2E намеренно блокирует основной bundle и проверяет русскую локализацию, кнопки и axe.
